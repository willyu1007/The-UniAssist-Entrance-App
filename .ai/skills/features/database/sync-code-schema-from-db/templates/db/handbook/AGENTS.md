# Database Handbook

## Purpose
Store database design notes and migration plans under `db/handbook/`.

## What belongs here
- Schema change proposals (desired state)
- Migration plans and verification checklists (human-run)
- Risk notes (backfill time, locking risk, rollback plan)

## What does NOT belong here
- Generated mirrors (do not edit `db/schema/tables.json`)
- Secrets/connection strings
